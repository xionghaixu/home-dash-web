# home-dash-web 前端页面规划与交互设计文档

> 文档定位：
> 本文件只承担前端页面规划、交互设计和阶段交付边界说明。
> 开发规范、组件编码规范、接口调用约束请查看 `SKILLS_SPEC.md`。
> README 保持项目介绍和运行说明，不与本文件混写。

## 1. 前端目标

`home-dash-web` 前端后续不再只是一个简单的文件管理后台，而是要逐步演进为：

- 个人网盘工作台
- 家庭 NAS 内容入口
- 图片、视频、音频的媒体中心
- 数据治理与设备联动界面

### 1.1 当前阶段约束

- 技术栈保持 Vue 3 + Vite + Element Plus + Pinia
- 当前仍以单体项目配套前端为主
- 默认面向单用户和家庭主人
- 第六阶段之前不做登录、权限、安全、域名和 SSL 相关页面
- 当前主线仍然是功能完善和页面完善，不引入复杂安全流程

### 1.2 文档边界

- 本文档讨论“页面做什么、交互怎么闭环、每个阶段交付哪些界面”
- `SKILLS_SPEC.md` 负责前端开发规范与工程约束
- 后端能力规划请查看 `../home-dash/OPTIMIZATION_SUGGESTIONS.md`

## 3. 页面规划原则

### 3.1 阶段独立

每个阶段都应该形成完整的页面闭环：

- 页面入口明确
- 数据结构稳定
- 空状态、加载状态、错误状态完整
- 关键弹窗、抽屉和详情面板一并交付
- 阶段结束后能形成一版可直接演示的前端产品形态

### 3.2 体验演进方向

页面体验应按如下路径演进：

- 阶段一到阶段二：以桌面优先的文件工作台为核心
- 阶段三：逐步转向内容浏览和家庭 NAS 入口体验
- 阶段四到阶段五：强化治理、任务、设备和备份管理能力

### 3.3 组件优先

后续页面建设应尽量沉淀为可复用组件，而不是散落在页面中：

- 列表型页面优先提炼表格、工具栏、空状态、错误状态组件
- 预览型页面优先提炼抽屉、弹层、播放器和详情组件
- 治理型页面优先提炼任务列表、统计卡片、筛选栏和图表容器

### 5.2 当前已完成页面与组件

- 已完成：系统信息页
- 已完成：视频播放页
- 已完成：404 页面
- 已完成：基础布局与路由切换

### 5.3 当前已完成交互

- 已完成：上传文件入口
- 已完成：新建文件夹
- 已完成：重命名
- 已完成：删除
- 已完成：移动
- 已完成：复制
- 已完成：下载
- 已完成：表格排序
- 已完成：目录切换
- 已完成：面包屑导航
- 已完成：视频文件跳转播放页

### 5.4 阶段一建议形成的一级菜单结构

当前只有“全部文件”和“系统信息”两个一级菜单，阶段一建议补齐为以下结构：

- 全部文件
- 最近上传
- 基础分类浏览
- 传输列表
- 系统信息

其中“基础分类浏览”在阶段一可以先做成轻量入口，不必直接进入完整媒体中心，但建议至少包含以下分类页或分类切换：

## 6. 阶段二：预览与检索整理

### 6.1 阶段目标

阶段二重点让用户“更容易找到文件、看见文件、理解文件”。

### 6.2 需要交付的页面和组件

- 搜索首页或搜索结果页
- 文件详情侧边栏
- 图片预览弹层
- 文本预览弹层
- 音频播放页或底部播放条
- 最近使用页
- 收藏页
- 最近搜索区域

### 6.3 页面能力规划

#### 搜索首页 / 搜索结果页

- 搜索输入区
- 最近搜索记录
- 筛选栏
- 排序栏
- 结果区
- 无结果状态
- 支持从搜索结果直接预览、收藏、跳转目录

建议支持以下筛选项：

- 文件类型
- 时间范围
- 大小区间
- 是否收藏
- 是否最近使用

#### 文件详情侧边栏

至少展示以下信息：

- 文件名
- 文件类型
- 文件大小
- 修改时间
- 所在目录
- 缩略图或预览入口
- 收藏按钮
- 关联操作入口

#### 预览能力

图片预览：

- 大图查看
- 上一张 / 下一张
- 缩放
- 原图下载

文本预览：

- 基础文本读取
- 长文本滚动查看
- 代码类文本保留基础等宽字体风格

音频预览：

- 播放 / 暂停
- 当前进度
- 总时长
- 最小化到底部播放条

#### 整理能力

- 收藏
- 最近使用
- 批量收藏

### 6.4 交互重点

- 预览不打断当前列表上下文
- 搜索结果和详情侧栏之间可以快速联动
- 收藏、最近使用能在多个页面中复用同一套交互
- 阶段二结束后，文件"查找"和"理解"体验要明显优于阶段一

### 6.5 阶段验收

- 搜索结果页可以独立承担查找入口
- 图片、文本、音频具备站内可用的基础预览体验
- 文件详情侧栏能成为统一的信息入口
- 最近使用、收藏形成基础整理闭环

## 7. 阶段三：媒体中心与家庭 NAS 体验

### 7.1 阶段目标

阶段三开始让前端真正具备家庭 NAS 入口的气质，让文件界面升级为内容界面。

### 7.2 需要交付的页面

- 首页工作台
- 图片库页
- 图片时间线页
- 视频库页
- 视频详情 / 增强播放页
- 音频库页
- 最近播放 / 最近浏览模块

### 7.3 建议形成的导航结构

阶段三之后，导航建议开始从“工作台 + 文件 + 系统”升级为“工作台 + 文件 + 媒体”的结构：

- 工作台
- 全部文件
- 图片
- 视频
- 音频
- 系统信息

### 7.4 页面能力规划

#### 首页工作台

首页建议成为内容入口，而不是单纯系统统计页。

建议模块：

- 最近上传
- 最近观看
- 继续播放
- 图片回顾
- 视频入口
- 音频入口
- 存储概况

#### 图片库页

- 网格视图
- 时间线视图
- 连续浏览
- 快速预览
- 多尺寸缩略图适配
- 按拍摄日期快速切换

#### 视频库页

- 封面
- 标题
- 时长
- 分辨率
- 最近播放时间
- 分类筛选
- 继续播放入口
- 横向卡片和网格卡片两种展示形式

#### 视频增强播放页

- 播放区域
- 继续播放提示
- 最近播放记录
- 关联文件信息

#### 音频库页

- 专辑视图
- 列表视图
- 最近播放
- 基础播放控制
- 歌手 / 专辑 / 目录切换

### 7.5 体验重点

- 阶段三页面应明显区别于“表格后台”气质
- 首页和媒体页要更偏内容浏览，而不是只显示数据表格
- 需要考虑大屏浏览和家庭电视场景的阅读性

### 7.6 阶段验收

- 首页可以承担家庭 NAS 内容入口
- 图片、视频、音频都有独立页面与独立浏览方式
- 继续播放等内容消费体验形成初版闭环
- 用户能从"文件管理"自然切换到"内容消费"模式

### 8.1 阶段目标

### 8.2 需要交付的页面

- 回收站页

### 8.3 页面能力规划

#### 回收站页

- 删除时间
- 原始目录
- 文件大小
- 恢复按钮
- 彻底删除按钮

#### 任务中心页

- 上传任务

- 重试按钮
- 来源说明

#### 存储分析页

- 容量占比
- 文件类型占比
- 最大目录排行
- 大文件排行

#### 重复文件页

- 重复组分组展示
- 建议保留项高亮
- 批量勾选与批量处理
- 原目录与大小对比
- 按目录范围筛选

#### 大文件页与清理建议页

- 大文件排行
- 目录占用排行
- 清理候选项
- 快速跳转目录
- 快速进入处理页

#### 整理建议页与巡检报告页

- 问题摘要卡片
- 巡检结果列表
- 失效预览资源提示
- 孤儿资源提示

#### 操作记录页

- 按时间倒序展示
- 按操作类型筛选
- 可跳转到关联目录或文件

- 页面需要清楚区分"内容浏览"和"治理操作"
- 任务状态必须易读，不应只剩状态码
- 图表与表格并存，确保既能看趋势也能做处理

### 8.5 阶段验收

- 用户可以自行恢复误删文件
- 用户可以知道哪些任务失败、为什么失败、如何重试
- 用户可以直观看到存储占用和可治理空间
- 用户可以直接执行部分治理动作

## 12. 当前推进建议

按照当前代码现状，前端优先级建议如下：

1. 先把阶段一的页面状态和反馈体验补齐，让现有文件工作台更稳。
2. 然后进入阶段二，优先做搜索页、详情侧栏和三类预览能力。
3. 阶段三再把首页、图片、视频、音频整体做成媒体中心体验。
4. 阶段四和阶段五分别承接治理界面和家庭设备联动界面。

这样可以最大化复用现有一阶段成果，也能让前端演进路径保持清晰。
